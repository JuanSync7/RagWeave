# Nginx HTTPS Gateway Design

**Date:** 2026-03-24
**Status:** Approved

## Problem

The RAG platform serves all traffic (API, web console, streaming queries) over plain HTTP on port 8000. This is unsuitable for:

- LAN demos where browsers flag insecure connections
- Future internet-facing demos requiring trusted HTTPS
- Production deployment readiness

## Decision

Add nginx as a TLS-terminating reverse proxy in front of FastAPI, running as a Compose service under a `gateway` profile (works with both Podman and Docker via `scripts/compose.sh`). Use `mkcert` for locally-trusted development certificates with a clean upgrade path to Let's Encrypt or tunnel services for internet access.

## Proxied Services

Only FastAPI (port 8000) is behind nginx. Ops/debug tools (Temporal UI, Grafana, Langfuse, Dozzle, Prometheus) remain direct-access on their existing ports.

## Nginx Configuration

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

# HTTP ‚Üí HTTPS redirect
server {
    listen 80;
    server_name aion.local localhost;
    return 301 https://$host$request_uri;
}

# HTTPS reverse proxy
server {
    listen 443 ssl;
    server_name aion.local localhost;

    # Generated by scripts/generate-certs.sh ‚Äî re-run if certs are missing
    ssl_certificate     /etc/nginx/certs/aion.local+1.pem;
    ssl_certificate_key /etc/nginx/certs/aion.local+1-key.pem;

    # Modern TLS settings
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # Proxy headers
    proxy_set_header Host              $host;
    proxy_set_header X-Real-IP         $remote_addr;
    proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # SSE support (buffering disabled below); WebSocket upgrade headers included for future use
    proxy_http_version 1.1;
    proxy_set_header Upgrade    $http_upgrade;
    proxy_set_header Connection $connection_upgrade;

    # Generous timeouts for LLM generation
    proxy_read_timeout 120s;
    proxy_send_timeout 120s;

    # Disable buffering for streaming responses
    proxy_buffering off;

    location / {
        proxy_pass http://rag-api:8000;
    }
}
```

### Key Config Decisions

- **SSE/streaming support**: `proxy_buffering off` is the critical directive for SSE. WebSocket upgrade headers are included for future use.
- **120s timeouts**: Matches the LLM generation budget (`RAG_STAGE_BUDGET_GENERATION_MS` is 60s; this gives headroom).
- **Forwarded headers**: FastAPI sees the real client IP and protocol via `X-Forwarded-For` / `X-Forwarded-Proto`.
- **Cert filenames**: `mkcert aion.local localhost` outputs `aion.local+1.pem` and `aion.local+1-key.pem` by convention.

## Compose Service

```yaml
rag-nginx:
  profiles: ["gateway"]
  image: nginx:alpine
  container_name: rag-nginx
  ports:
    - "${RAG_NGINX_HTTP_PORT:-80}:80"
    - "${RAG_NGINX_HTTPS_PORT:-443}:443"
  volumes:
    - ./ops/nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
    - ./certs:/etc/nginx/certs:ro
  depends_on:
    rag-api:
      condition: service_healthy
  healthcheck:
    test: ["CMD-SHELL", "wget --no-check-certificate --spider -q https://localhost/health || exit 1"]
    interval: 10s
    timeout: 3s
    retries: 5
  restart: unless-stopped
```

- **`gateway` profile**: Nginx only starts when explicitly opted in (`--profile gateway`).
- **Depends on `rag-api` healthcheck**: Avoids 502 errors on startup.
- **Healthcheck**: Verifies nginx is serving HTTPS and can reach the backend.
- **Ports configurable**: Defaults to 80/443, overridable via `.env`.
- **Read-only mounts**: Nginx cannot modify config or certs.
- **`nginx:alpine`**: Small image (~7MB), Podman-compatible.
- **Networking**: Nginx resolves `rag-api` by service name via the default compose network. No custom network configuration is needed.

### Usage

```bash
# Local dev with gateway
./scripts/compose.sh --profile app --profile gateway up -d

# Full stack
./scripts/compose.sh --profile app --profile workers --profile gateway up -d
```

The `gateway` profile only contains nginx. You must also activate the `app` profile to start the services nginx depends on (`rag-api`, `rag-redis`, etc.). Running `--profile gateway` alone will hang at startup waiting for `rag-api` healthcheck.

## Certificate Generation

### `scripts/generate-certs.sh`

```bash
#!/usr/bin/env bash
# Generate locally-trusted TLS certs for aion.local + localhost
set -euo pipefail

CERT_DIR="$(cd "$(dirname "$0")/.." && pwd)/certs"
mkdir -p "$CERT_DIR"

# Install mkcert root CA if not already done
mkcert -install

# Generate cert for both hostnames
mkcert -cert-file "$CERT_DIR/aion.local+1.pem" \
       -key-file  "$CERT_DIR/aion.local+1-key.pem" \
       aion.local localhost 127.0.0.1 ::1

echo "Certs written to $CERT_DIR/"
echo ""
echo "Add to /etc/hosts if not already present:"
echo "  127.0.0.1  aion.local"
```

### Setup Flow

1. **Install mkcert** (one-time): `sudo apt install mkcert` or `brew install mkcert`
2. **Generate certs** (one-time): `./scripts/generate-certs.sh`
3. **Add hosts entry** (one-time): `echo "127.0.0.1  aion.local" | sudo tee -a /etc/hosts`
4. **Start the stack**: `./scripts/compose.sh --profile app --profile gateway up -d`
5. **Browse**: `https://aion.local`

## FastAPI Changes

The only change to FastAPI is updating the uvicorn command in `docker-compose.yml` to trust proxy headers:

```yaml
command: uvicorn server.api:app --host 0.0.0.0 --port ${RAG_API_PORT:-8000} --proxy-headers --forwarded-allow-ips='*'
```

This ensures:
- `request.url` reflects `https://` when accessed through nginx
- `X-Forwarded-For` is trusted for client IP logging
- No application code changes required

### Security Note: Direct Port Exposure

When the `gateway` profile is active, `rag-api` port 8000 remains exposed to the host by default (from the `app` profile). This means the API is accessible both via `https://aion.local` (nginx) and `http://localhost:8000` (direct, bypassing TLS).

**`--forwarded-allow-ips='*'`** trusts `X-Forwarded-For` from any source, meaning a client hitting port 8000 directly could spoof the forwarded IP header.

For local dev this is acceptable. For LAN demos or hardened deployments, restrict direct API access by setting in `.env`:

```
# Bind rag-api to loopback only ‚Äî only nginx (via compose network) can reach it
RAG_API_HOST_PORT=127.0.0.1:8000
```

This ensures external LAN clients must go through nginx (HTTPS) while localhost tooling (curl, CLI client) still works.

## Future Internet Access

When internet-facing access is needed:

- **Option A ‚Äî Real domain + Let's Encrypt**: Buy a domain, point DNS, add certbot sidecar or swap certs. Nginx config stays the same.
- **Option B ‚Äî Tunnel service**: Add Cloudflare Tunnel or Tailscale alongside the stack. Zero cert management, free tiers available. Fastest path for one-off demos.

Both options work with this nginx setup ‚Äî only the cert source changes.

## Summary

| Aspect | Decision |
|--------|----------|
| **Proxy** | nginx (alpine) as compose service, `gateway` profile |
| **TLS** | mkcert for local dev, swap-ready for Let's Encrypt |
| **Hostnames** | `aion.local` + `localhost` |
| **Ports** | 443 (HTTPS), 80 (redirect to HTTPS) |
| **Proxied service** | FastAPI only (`rag-api:8000`) |
| **Streaming** | Buffering disabled, WebSocket/SSE headers, 120s timeouts |
| **FastAPI changes** | `--proxy-headers --forwarded-allow-ips='*'` on uvicorn command |
| **Future internet** | Tunnel service or real domain + Let's Encrypt |
