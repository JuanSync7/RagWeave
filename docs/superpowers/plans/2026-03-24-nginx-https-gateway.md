# Nginx HTTPS Gateway Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add nginx as a TLS-terminating reverse proxy in front of FastAPI with mkcert certificates, running as a Compose service under a `gateway` profile.

**Architecture:** Nginx container terminates TLS on ports 80/443 and proxies to `rag-api:8000` over plain HTTP inside the compose network. Certs are generated locally with `mkcert` and mounted read-only. No FastAPI application code changes — only the uvicorn command gains `--proxy-headers`.

**Tech Stack:** nginx:alpine, mkcert, Podman/Docker Compose, bash

**Spec:** `docs/superpowers/specs/2026-03-24-nginx-https-gateway-design.md`

### Task 3: Create nginx configuration

**Files:**
- Create: `ops/nginx/nginx.conf`

- [ ] **Step 1: Create ops/nginx directory**

```bash
mkdir -p ops/nginx
```

- [ ] **Step 2: Write nginx.conf**

Create `ops/nginx/nginx.conf`:

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

# HTTP → HTTPS redirect
server {
    listen 80;
    server_name aion.local localhost;
    return 301 https://$host$request_uri;
}

# HTTPS reverse proxy
server {
    listen 443 ssl;
    server_name aion.local localhost;

    # Generated by scripts/generate-certs.sh — re-run if certs are missing
    ssl_certificate     /etc/nginx/certs/aion.local+1.pem;
    ssl_certificate_key /etc/nginx/certs/aion.local+1-key.pem;

    # Modern TLS settings
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # Proxy headers
    proxy_set_header Host              $host;
    proxy_set_header X-Real-IP         $remote_addr;
    proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # SSE support (buffering disabled below); WebSocket upgrade headers included for future use
    proxy_http_version 1.1;
    proxy_set_header Upgrade    $http_upgrade;
    proxy_set_header Connection $connection_upgrade;

    # Generous timeouts for LLM generation
    proxy_read_timeout 120s;
    proxy_send_timeout 120s;

    # Disable buffering for streaming responses
    proxy_buffering off;

    location / {
        proxy_pass http://rag-api:8000;
    }
}
```

### Task 4: Create certificate generation script

**Files:**
- Create: `scripts/generate-certs.sh`

- [ ] **Step 1: Write generate-certs.sh**

Create `scripts/generate-certs.sh`:

```bash
#!/usr/bin/env bash
# Generate locally-trusted TLS certs for aion.local + localhost
set -euo pipefail

CERT_DIR="$(cd "$(dirname "$0")/.." && pwd)/certs"
mkdir -p "$CERT_DIR"

if ! command -v mkcert &>/dev/null; then
    echo "Error: mkcert is not installed."
    echo "Install with: sudo apt install mkcert (Debian/Ubuntu) or brew install mkcert (macOS)"
    exit 1
fi

# Install mkcert root CA if not already done
mkcert -install

# Generate cert for both hostnames
mkcert -cert-file "$CERT_DIR/aion.local+1.pem" \
       -key-file  "$CERT_DIR/aion.local+1-key.pem" \
       aion.local localhost 127.0.0.1 ::1

echo ""
echo "Certs written to $CERT_DIR/"
echo ""
echo "Add to /etc/hosts if not already present:"
echo "  127.0.0.1  aion.local"
```

```yaml
    profiles: ["gateway"]
    image: nginx:alpine
    container_name: rag-nginx
    ports:
      - "${RAG_NGINX_HTTP_PORT:-80}:80"
      - "${RAG_NGINX_HTTPS_PORT:-443}:443"
    volumes:
      - ./ops/nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - ./certs:/etc/nginx/certs:ro
    depends_on:
      rag-api:
        condition: service_healthy
    healthcheck:
      test: ["CMD-SHELL", "wget --no-check-certificate --spider -q https://localhost/health || exit 1"]
      interval: 10s
      timeout: 3s
      retries: 5
    restart: unless-stopped

```
