### `src/db/minio/store.py` — MinIO Document and Page Image Store

**Purpose:**

This module provides all low-level MinIO object storage operations used by the RAG system. It covers two distinct concerns: (1) text document storage — put, get, delete, exists-check, presigned URL generation, and listing of markdown content objects and their JSON metadata sidecars; and (2) page image storage — storing JPEG page images produced by the visual embedding pipeline, deleting them, and generating presigned GET URLs for retrieved page images. The page image functions (`store_page_images`, `delete_page_images`, `get_page_image_url`) are used exclusively by the visual pipeline. All other functions serve the core document management path.

Spec requirements addressed: FR-401 (page image key pattern), FR-402 (JPEG quality), FR-403 (presigned URL generation with configurable expiry and bucket), FR-404 (delete by prefix), FR-405 (pre-storage cleanup in update mode), FR-607 (presigned URL for retrieval results).

**How it works:**

**Document storage (core path):**

1. `create_client()`: Constructs a `Minio` client from five settings: `MINIO_ENDPOINT`, `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY`, `MINIO_BUCKET`, `MINIO_SECURE`. The client is stateless and safe to share across threads.

2. `put_document(client, document_id, content, metadata, bucket)`: Writes two objects to MinIO — `{document_id}.md` (UTF-8 markdown) and `{document_id}.meta.json` (JSON-encoded metadata dict). Both are written unconditionally, overwriting any existing objects.

3. `get_document(client, document_id, bucket)`: Fetches `{document_id}.md`; returns `None` if `S3Error.code` is `"NoSuchKey"` or `"NoSuchBucket"`. Attempts to fetch the `.meta.json` sidecar; a missing sidecar is non-fatal. Returns a dict with `document_id`, `content`, `metadata` keys.

4. `get_document_url(client, document_id, bucket, expires_in_seconds)`: Calls `client.presigned_get_object(bucket, f"{document_id}.md", expires=timedelta(seconds=expires_in_seconds))`. Appends the `.md` suffix automatically.

5. `build_document_id(source_key)`: Returns `str(uuid.uuid5(uuid.NAMESPACE_URL, f"doc:{source_key}"))` — a deterministic UUID stable across ingestion runs.

**Page image storage (visual pipeline path):**

6. `store_page_images(client, document_id, pages, quality, bucket)`: For each `(page_number, PIL.Image)` tuple in `pages`:
   - Computes key: `pages/{document_id}/{page_number:04d}.jpg` (FR-401).
   - Encodes the image to JPEG in a `BytesIO` buffer at the specified `quality` (FR-402).
   - Seeks to measure the buffer length, resets to start, then calls `client.put_object(bucket, key, buffer, length=length, content_type="image/jpeg")`.
   - On per-page failure, logs WARNING and continues. The failed key is not added to `stored_keys`.
   - Returns list of keys that were successfully stored.

7. `delete_page_images(client, document_id, bucket)`: Lists all objects under prefix `pages/{document_id}/` and removes them one by one. Per-object removal failures are logged and cause early return (remaining objects are not attempted). Returns the count of deleted objects.

8. `get_page_image_url(client, minio_key, bucket, expires_in_seconds)`: Unlike `get_document_url`, this function uses `minio_key` as-is — no suffix is appended. If `bucket` is empty, falls back to `MINIO_BUCKET`. If `expires_in_seconds` is `0`, falls back to `RAG_VISUAL_RETRIEVAL_URL_EXPIRY_SECONDS`. The presigned URL is generated by `client.presigned_get_object(bucket, minio_key, expires=timedelta(seconds=expires_in_seconds))`. MinIO does not validate object existence at signing time — a URL for a non-existent key is still generated and will return HTTP 404 when accessed.

```python
# Key pattern and JPEG encoding in store_page_images
key = f"pages/{document_id}/{page_number:04d}.jpg"
buffer = io.BytesIO()
image.save(buffer, format="JPEG", quality=quality)
buffer.seek(0, 2)       # seek to end to measure
length = buffer.tell()
buffer.seek(0)          # reset before upload
client.put_object(bucket, key, buffer, length=length, content_type="image/jpeg")
```

**Key design decisions:**

| Decision | Alternatives Considered | Why This Choice |
|----------|------------------------|-----------------|
| Key pattern `pages/{document_id}/{page_number:04d}.jpg` | Flat namespace, content-hash keys | Prefix-based namespace allows efficient bulk listing and deletion by document via `list_objects(prefix=...)`. Zero-padded page numbers ensure lexicographic order matches page order. |
| Sentinel defaults (`bucket=""`, `expires_in_seconds=0`) for `get_page_image_url` | Explicit `None` defaults | Python's module-level constant `MINIO_BUCKET` is resolved at import time. Using sentinel values allows deferred resolution inside the function body, keeping the public signature clean while allowing the function's defaults to track runtime config changes. |
| Per-page upload isolation (continue on failure) | Abort on first failed page | Documents with occasional corrupt or oversized pages should still produce partial results. Callers receive the list of successfully stored keys and can track coverage from the count. |
| Separate `get_page_image_url` from `get_document_url` | Unify into one function with a flag | Page images use raw keys (no suffix appended); documents use constructed keys (suffix appended). Combining them would require a flag or conditional that would obscure the different semantics. Two explicit functions are clearer. |
| JPEG format for page images | PNG, WebP | JPEG provides the best balance of file size and decode speed for photographic page scans. Quality=85 (default) retains sufficient visual fidelity for human review at ~60–80% smaller file size than PNG. |

**Configuration:**

| Parameter | Type | Default | Effect |
|-----------|------|---------|--------|
| `MINIO_ENDPOINT` | `str` | `"localhost:9000"` (env: `RAG_MINIO_ENDPOINT`) | MinIO server host:port |
| `MINIO_ACCESS_KEY` | `str` | `"minioadmin"` (env: `RAG_MINIO_ACCESS_KEY`) | MinIO access credential |
| `MINIO_SECRET_KEY` | `str` | `"minioadmin"` (env: `RAG_MINIO_SECRET_KEY`) | MinIO secret credential |
| `MINIO_BUCKET` | `str` | `"rag-documents"` (env: `RAG_MINIO_BUCKET`) | Default bucket for all operations |
| `MINIO_SECURE` | `bool` | `False` (env: `RAG_MINIO_SECURE`) | Use TLS for MinIO connection |
| `quality` (store_page_images) | `int` | `85` (from `RAG_INGESTION_PAGE_IMAGE_QUALITY`) | JPEG compression quality 1–100 |
| `expires_in_seconds` (get_page_image_url) | `int` | `0` → resolved to `RAG_VISUAL_RETRIEVAL_URL_EXPIRY_SECONDS` (default 3600) | Presigned URL TTL in seconds |

**Error behavior:**

- `S3Error` with code `"NoSuchKey"` or `"NoSuchBucket"`: `get_document` returns `None`; not re-raised.
- `S3Error` with any other code from `get_document`: Re-raised to caller.
- Per-page upload failure in `store_page_images`: Logged as WARNING; the page is skipped and `store_page_images` continues. The returned list omits the failed key.
- Per-object deletion failure in `delete_page_images`: Logged as WARNING; causes early return. The return value reflects only the count of objects deleted before the failure.
- Listing failure in `delete_page_images`: Logged as WARNING; returns `0`.
- `get_page_image_url` with a non-existent key: No error at signing time. The resulting URL will return HTTP 404 when accessed by a browser or client. The caller (RAGChain) does not validate URL accessibility.
